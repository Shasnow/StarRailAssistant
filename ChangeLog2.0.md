### 更新说明：

#### 破坏性变更：
- 移除了启动命令run和single。
  - 我们在2.11版本中已将这两个命令标记为过时，现在它们已被移除。您可以使用`-e`选项或使用启动命令来替代。
  - 示例：
  - 旧命令：SRA-cli run --config Default --once
  - `-e`选项：SRA-cli -e run Default+exit
  - 启动命令：SRA-cli "run Default" exit
- 移除了`--embed`选项，请使用`--inline`选项

#### 启动命令
- SRA-cli 支持在调用应用时通过将命令作为额外参数传递给程序来发送命令。
- 示例：SRA-cli "run Default" exit
- SRA-cli 将每个参数解释为一个单独的命令，因此如果命令超过一个单词，你应该将每个命令用引号括起来。你可以使用单引号或双引号。
- 这将在启动SRA后立即阻塞式运行“Default”配置文件，并在完成后执行“exit”命令退出SRA。

#### 启动脚本
- SRA-cli 支持启动脚本，您可以在当前目录下创建一个名为`.srarc`的文件，用于在启动时自动执行的命令。
- 示例：
  ```
  run Default
  exit
  ```
  - 这将在启动SRA后立即阻塞式运行“Default”配置文件，并在完成后执行“exit”命令退出SRA。
  - 您可以在启动脚本中添加多个命令，每个命令占一行。命令将按顺序执行，直到遇到“exit”命令或脚本结束。

#### 货币战争刷开局投资策略支持同一位面设置多个策略
- 此前，货币战争刷开局投资策略只能设置一个策略，这对于一些位面来说可能不够灵活。现在，您可以在同一位面设置多个策略，只要满足其中一个策略的条件，就会执行下一步操作。
- 例如：“白银投资 溜佩佩 招财狗；”表示第一位面的投资策略出现“白银投资”或“溜佩佩”或“招财狗”中的任意一个即可执行下一步操作。
