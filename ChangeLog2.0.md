#### 后端启动参数
- 后端启动参数是指在启动后端进程时传递的参数，可以用来调整后端的行为。例如，可以使用`run --config Default`来使启动时立即运行默认配置。
- 在控制台的高级设置中，可以添加自定义的后端启动参数，这些参数会在每次启动后端时自动传递给后端进程。
- 默认含有`--inline`参数，建议保留该参数以确保后端正常运行，如果需要添加其他参数，可以在此参数前面添加新的参数。
- 例如，如果需要在启动时立即运行默认配置，可以将启动参数设置为`run --config Default --inline `
- 更多后端启动参数可以参考文档

#### 传递启动参数
- 前端现在支持向后端传递启动参数，下面是一些示例：
    ```bash
     SRA.exe run --config Default
    ```
  等价于
    ```bash
     SRA-cli.exe run --config Default
    ```
  区别在于前者会启动前端界面并获取完整支持（例如自动获取游戏路径、自动设置分辨率等），因此更建议使用前者。
