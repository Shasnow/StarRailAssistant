# SRA已更新到 2.x
## Full Changelog：
- 更新日期：
- 版本：2.11

### 主要更新内容:
- 货币战争支持自动应用攻略码。
- 新增Webhook、Telegram、Server 酱、OneBot等10个外部通知渠道。
- 现在可以为每个任务单独配置通知时机，例如在清体力任务开始、结束时发送通知。
- 控制台新增“停止后端”按钮，可直接停止后端进程。
- 新增高级设置-->后端启动参数，可自定义后端启动参数。
- 前端支持向后端传递启动参数。
- SRA CLI工具新增“--execute”启动参数，可在启动时直接执行指定的命令。多个命令可以使用`&`符号分隔。
- 新增“--use-python”参数，可以临时启用Python后端。
- 新增游戏叠加层功能，可以在游戏界面上显示SRA的运行状态和相关信息。
- 开发者选项之--在游戏叠加层显示调试信息，可以在游戏界面上显示更多的调试信息（例如鼠标相对位置），帮助开发者进行调试和优化。

### 功能调整：
- 攻略字段“name”已调整为“title”，请注意更新相关配置文件。
- 后端启动参数“run”和“single”已被标记为过时，建议使用新的参数格式，例如`--execute "run Default"`来替代`run --config Default`。
- 差分宇宙“刷单层”功能已重命名为“刷第一关”，功能保持不变。

### 问题修复：
- 修复了存在僵尸进程时误判游戏正在运行的问题。
- 修复了货币战争结束后未退出到大世界界面导致的后续任务无法正常运行的问题。

### 更新说明：
#### 即将到来的破坏性变更：
- 后端启动参数“run”和“single”将在未来版本中被移除，请尽快更新相关用法。
- 示例：
    - 旧用法：`run --config Default`
    - 新用法：`--execute "run Default"`
      -  或；`-e "run Default"`
    - 旧用法：`run --config Default --once`（仅运行一次后退出）
    - 新用法：`--execute "run Default & exit"`
      -  或：`-e "run Default & exit"`
- `--execute`与`-e`,`--command`,`-c`为相同的参数

#### 外部通知渠道
- SRA现在支持Webhook、Telegram、Server 酱、OneBot等10个外部通知渠道，用户可以在设置中配置这些通知渠道，以便在特定事件发生时接收通知。
- 查看文档了解如何配置和使用这些通知渠道。

#### 货币战争支自动应用攻略码
- 攻略文件新增“share_code”字段，用于指定攻略码，如果该字段缺失或为空字符串，则不应用攻略码。
- 只需在攻略配置中添加相应的攻略码，SRA会在开局时自动应用这些攻略码，以便锁定角色装备。

#### 后端启动参数
- 后端启动参数是指在启动后端进程时传递的参数，可以用来调整后端的行为。
- 在控制台的高级设置中，可以添加自定义的后端启动参数，这些参数会在每次启动后端时自动传递给后端进程。
- 默认含有`--inline`参数，建议保留该参数以确保后端正常运行，如果需要添加其他参数，可以在此参数前面添加新的参数。
- 例如，如果需要在应用启动时立即运行默认配置，可以将启动参数设置为`--execute "task run Default" --inline `。
- 例如，在单独启动后端时，如果要在启动后立即运行默认配置，并在完成后退出，可以使用以下命令：
    ```bash
     SRA-cli.exe --execute "run Default & exit"
    ```
   或者
    ```bash
     SRA-cli.exe -e "run Default & exit"
    ```
- 更多后端命令可以参考文档。

#### 传递启动参数
- 前端现在支持向后端传递启动参数，下面是一些示例：
    ```bash
     SRA.exe --execute "task run Default"
    ```
  等价于
    ```bash
     SRA-cli.exe --execute "task run Default"
    ```
  区别在于前者会启动前端界面并获取完整支持（例如自动获取游戏路径、自动设置分辨率等），因此更建议使用前者。
